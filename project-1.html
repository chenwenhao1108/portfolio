OneMore

OneMore is a CSR single page flashcard Web app with user authentication functionality where any user can create their own decks to collect all the cards they want to memorize. And the the card will show up according to the user's content of memory to the very card.

Live Link

Project Overview

The OneMore app is a real-world web app used for user to create their very own deck, and user can create their own card and collect it to the deck that they created, using compound component and custom hook build reusable component.

And the logic behind the app will calculate when is the time to show up the card again. The result of the calculation won't be uploaded to the firestore immediately for performance consideration, instead being uploaded after the user finishing this run of learning.

Users can design the style of their deck and cards as they want. They can upload the cover to the deck. When creating a card, the content of the card could include 4 levels of heading, 5 colors of fonts, and quote content or code section and so on.

And tiptap as the rich text editor for users to create their cards or show their cards so that the cards can easily hold their styles and the users can also easily re-edit the card if they want.

The app is built by React.js and added React Router 6 to create single page routing, using Context API to manage states of the app, useMemo and React.memo() to reduce unnecessary re-calculation or re-render to enhance performance.

Using Google firebase firestore to hold all the data of the users, like their decks and cards or usernames and avatars.Google firebase Authentication to authenticate the users, for users to register and login.

Using GitHub to controlling the versions, and vite to pack the project, Netlify to deploy the app
